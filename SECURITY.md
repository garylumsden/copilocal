# Security Policy

## Supported versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| Older releases | No |

## Reporting a vulnerability

If GitHub private vulnerability reporting is enabled, please use the repository's private security advisory flow.

If not, open a GitHub issue with a short description and label/title prefix `security`. Do not include exploit details, secrets, or private data in a public issue. A maintainer will follow up to coordinate details privately.

We aim to acknowledge reports within 7 days.
